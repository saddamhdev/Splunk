## How Elasticsearch works (simple)

1. **Data ‚Üí JSON documents**
2. Stored in **Indexes**
3. Indexes split into **Shards**
4. Shards distributed across **Nodes**
5. Uses **inverted index** ‚Üí lightning-fast search

Think:

> Table = Index
> Row = Document
> Column = Field

## Example (very real)

Your Spring Boot app logs:

```json
{
  "service": "user-service",
  "status": 500,
  "error": "NullPointerException",
  "time": "2026-01-23T12:01:00"
}
```

Elasticsearch lets you:

* Find all `500` errors
* Group by service
* See spikes over time
* Build dashboards in Kibana

## Elasticsearch vs Splunk (quick clarity)

| Topic       | Elasticsearch      | Splunk                  |
| ----------- | ------------------ | ----------------------- |
| Type        | Search engine      | Full analytics platform |
| Cost        | Open-source (core) | Expensive (licensed)    |
| Flexibility | Very high          | High but opinionated    |
| Learning    | Moderate           | Easier for non-devs     |
| SIEM        | Possible           | Excellent out of box    |

* JVM-heavy
* You manage infra unless using Elastic Cloud
